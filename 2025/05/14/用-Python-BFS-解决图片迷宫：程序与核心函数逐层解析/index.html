用 Python BFS 解决图片迷宫：程序与核心函数逐层解析

本文整理一个用 Python 解决图片迷宫的程序。程序的核心思路是：把迷宫图片看成由像素点组成的网格，白色像素表示可以通行，非白色像素表示墙壁或障碍；然后使用广度优先搜索（Breadth-First Search，简称 BFS）从起点出发寻找终点；找到路径后，再把路径像素标记为红色并保存为新的图片。

为了便于理解，本文按照下面的顺序展开：

  1. 先看完整程序。
  2. 再理解程序的整体流程。
  3. 然后逐个解释核心函数：iswhite、getadjacent、BFS。
  4. 再重点解释 BFS 中用到的关键数据结构：came_from、deque、set、len()。
  5. 最后补充代码细节、常见问题和路径图像输出。

图像迷宫如何转化为搜索问题

这段程序的关键，是把“图片迷宫”转化成“图搜索问题”。

在程序中：

  • 每个像素点可以看作一个节点。
  • 白色像素表示可以走。
  • 非白色像素表示墙壁或障碍物。
  • 每个像素最多有四个相邻像素：左、上、右、下。
  • 从起点出发，只要不断访问相邻的白色像素，就能在图中搜索路径。

也就是说，迷宫图片本质上变成了一个巨大的网格图，而 BFS 就是在这个网格图上寻找从 start 到 end 的路径。

BFS 的关键特性

BFS 会找到最短路径之一

BFS 的搜索方式是从起点开始，一层一层向外扩展。

它会先探索距离起点为 1 的位置，再探索距离起点为 2 的位置，然后再探索距离起点为 3 的位置。由于每次移动的代价相同，所以 BFS 找到的路径一定是最短路径之一。

came_from 用于路径回溯

BFS 搜索本身只负责“走到哪里”。如果要还原完整路径，就必须记录每个位置是从哪里来的。

为什么 BFS 中使用 deque？

主要原因是高效。

deque 的 append() 和 popleft() 操作都是 O(1) 时间复杂度。

如果用列表 list 实现队列，pop(0) 需要移动后面的所有元素，因此是 O(n) 操作。

操作 list deque
append O(1) O(1)
appendleft O(n) O(1)
pop O(1) O(1)
popleft O(n) O(1)

除了高效以外，deque 还有两个特点：

  • append() 和 popleft() 是原子操作，在多线程场景下有一定安全性。
  • 可以方便地在两端添加或移除元素，使用灵活。

什么时候使用 set，什么时候使用 list？

适合使用 set() 的情况：

  • 需要快速判断某个元素是否存在。
  • 需要去除重复元素。
  • 需要执行集合运算，例如并集、交集、差集等。

适合使用 list 的情况：

  • 需要保持元素顺序。
  • 允许重复元素。

总结

这段程序的核心可以概括为一句话：

将迷宫图片看成一个由像素组成的网格图，用 BFS 在白色像素之间寻找从起点到终点的最短路径，然后把路径像素改成红色并保存图片。

其中最重要的几个概念是：

  • iswhite：判断像素是否是可通行的白色区域。
  • getadjacent：获取当前像素的上下左右四个相邻位置。
  • deque：实现 BFS 队列，保证先进先出。
  • visited：记录已经访问过的位置，避免重复搜索。
  • came_from：记录路径来源，用于从终点反向回溯到起点。
  • len(queue) > 0：判断 BFS 队列中是否还有待探索的位置。
  • pixels[x, y] = (255, 0, 0)：将路径像素标记为红色。

从算法角度看，BFS 的优势是：当每一步移动的代价相同时，它能够找到从起点到终点的最短路径之一。
