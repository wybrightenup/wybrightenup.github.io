无尽边界：Mandelbrot 集之美

Posted on 2026-05-16

很多人第一次看到 Mandelbrot 集时，都会被它的形状震住：

  • 远看像一只黑色的“心形甲虫”；
  • 近看边界卷曲、分叉、重复；
  • 再继续放大，新的结构还会不断冒出来。

它看起来非常神秘，但如果我们站在“程序是怎么画出它的”这个角度去看，事情其实很清楚：

  1. 先在复平面上取很多点；
  2. 对每个点做同样的迭代；
  3. 根据这个点“逃逸得有多快”给它上色；
  4. 最后把结果保存成一张图片。

这篇文章的目标，不是只给出一个能运行的 Mandelbrot 程序，而是把这段程序背后的每一步都讲透，尤其是最关键也最容易让人迷惑的一步：

程序到底是怎么在复平面里生成一整张图所需要的那些点的？

这一步理解了，Mandelbrot 集的程序就算真正入门了。

Mandelbrot 集到底是什么？

Mandelbrot 集研究的是下面这个迭代：

\[ z_{n+1} = z_n^2 + c, \qquad z_0 = 0 \]

这里：

  • c 是复平面上的一个点；
  • z 是从 0 开始不断迭代出来的复数序列。

我们关心的问题只有一个：

对于某个给定的复数 c，从 z=0 开始不断迭代，z 会不会越跑越远？

如果它很快越跑越大，说明这个点在 Mandelbrot 集外面。 如果它怎么迭代都不明显发散，我们就把它视为 Mandelbrot 集内部，或者至少非常接近边界。

于是，绘制 Mandelbrot 图，本质上就是：

把复平面切成很多很多个点，对每个点都问一遍：你会不会逃逸？

程序最终画出来的，并不是一个“神秘的形状”，而是一个巨大的“判定结果图”。

图像像素和复平面点之间的对应关系

一张图像是一个矩形网格：

  • 横向有 width 个像素；
  • 纵向有 height 个像素。

复平面中的观察区域，也可以看成一个矩形：

  • 横向对应复数的实部；
  • 纵向对应复数的虚部。

所以最自然的映射就是：

  • 图像每一列，对应一个实部；
  • 图像每一行，对应一个虚部；
  • 行和列一组合，就得到一个复数 c = 实部 + 虚部 * i。

这就是“图像采样”的本质。
